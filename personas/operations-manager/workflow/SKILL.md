# 运营经理工作流

## 工作流全景

```
运营诊断 → 流程优化 → OKR拆解 → 增长执行 → 数据复盘 → 规模化复制
    ↓           ↓          ↓           ↓           ↓           ↓
  现状摸底   SOP建立    指标体系    活动执行    周期复盘    方法论沉淀
```

---

## 完整工作流

### 阶段一：运营现状诊断

**触发**：新接手业务 / 季度开始 / 老板要看运营效率

```
1. 加载COO框架
   skill_view("ops/coo-advisor")
   → 关键问题：
     - 哪些流程是瓶颈？
     - 哪些环节人工>30%时间？
     - 数据断层在哪里？

2. 绘制当前运营地图：
   - 核心流程 × 负责人 × SLA
   - 识别高频问题点
```

---

### 阶段二：流程优化与SOP建立

**触发**：发现重复性问题 / 新员工频繁犯同类错误

```
1. 加载流程框架
   skill_view("ops/process-frameworks")
   → SIPOC分析（供应商-输入-流程-输出-客户）
   → 5Why根因分析

2. SOP撰写原则：
   - 用动词开头的步骤
   - 明确输入条件和验收标准
   - 异常处理分支

3. 写入 TASKLOG.md：
   | SOP-[流程名] | 完成 | [日期] | 预计节省Xh/周 |
```

---

### 阶段三：增长运营

**触发**：新活动上线 / 增长目标制定 / 渠道ROI分析

```
1. 加载增长框架
   skill_view("ops/growth-metrics")
   → 增长飞轮：拉新→激活→留存→变现→传播
   → 关键指标定义

2. 活动执行三要素：
   - 清晰的北极星指标
   - A/B测试设计
   - 复盘节点预设

3. 数据追踪：每周对比目标 vs 实际
```

---

### 阶段四：运营节奏管理

**触发**：团队缺乏节奏感 / 信息同步成本高

```
1. 加载运营节奏框架
   skill_view("ops/ops-cadence")
   → 日会/周会/月会/季度会议设计
   → 汇报模板标准化

2. 信息分发原则：
   - 日报：执行层（今天做了什么/阻塞）
   - 周报：管理层（进展/风险/下周计划）
   - 月报：决策层（目标达成/趋势/资源需求）
```

---

### 阶段五：规模化复制

**触发**：一个地区/品类跑通了，要向更多地方复制

```
1. 加载规模化框架
   skill_view("ops/scaling-playbook")
   → 标准化vs本地化的边界
   → 复制失败的常见原因

2. 复制前必做清单：
   - 成功要素是否已提炼？
   - 依赖的关键资源是否可复制？
   - 指标定义在新场景是否适用？
```
