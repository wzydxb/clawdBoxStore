# HR经理工作流

## 工作流全景

```
人才需求 → 招聘执行 → 入职融入 → 绩效管理 → 薪酬激励 → 组织发展 → 文化建设
    ↓           ↓          ↓           ↓           ↓           ↓           ↓
  JD撰写    面试框架    Onboarding  绩效评估   薪酬对标    组织设计    留存策略
```

---

## 完整工作流

### 阶段一：招聘需求分析

**触发**：业务团队提出招聘需求

```
1. 加载HR战略框架
   skill_view("hr/chro-advisor")
   → 核实：这个岗位现在是否真正需要？能否用外包/工具替代？

2. 三个关键问题：
   - 这个岗位解决什么具体问题？
   - 6个月后如何衡量该员工成功？
   - 预算和周期是否匹配？

3. 撰写JD：明确职责、要求、成功标准
```

---

### 阶段二：招聘与面试

**触发**：JD发布，候选人进入流程

```
1. 设计面试框架
   skill_view("hr/chro-advisor")
   → 结构化面试题（行为/情境/专业）
   → 评分维度定义

2. 候选人评估标准：
   - 技能匹配度（硬技能）
   - 文化契合度（价值观）
   - 成长潜力（软技能）

3. 写入 TASKLOG.md：
   | 招聘-[职位] | 进行中 | [日期] | 共N位候选人 |
```

---

### 阶段三：绩效与薪酬

**触发**：季度绩效周期 / 调薪 / offer设计

```
1. 薪酬对标分析
   skill_view("hr/comp-frameworks")
   → 市场薪酬分位数（P50/P75/P90）
   → 输出：薪酬区间建议

2. 绩效评估设计
   skill_view("hr/people-strategy")
   → OKR/KPI设置原则
   → 绩效面谈话术

3. 高风险信号：薪酬低于市场20%以上 → 主动提示留存风险
```

---

### 阶段四：组织发展

**触发**：公司扩张 / 业务调整 / 管理层变动

```
1. 组织架构设计
   skill_view("hr/org-design")
   → 汇报关系优化
   → 团队层级建议（扁平vs层级）

2. 人才梯队规划：
   - 识别关键岗位（离职则业务中断）
   - 制定继任者计划
   - 高潜人才发展路径

3. 写入 TASKLOG.md
```
