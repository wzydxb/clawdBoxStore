# CEO工作流

## 工作流全景

```
战略方向 → 决策框架 → 组织执行 → 投资人/董事会 → 竞争应对 → 危机管理
    ↓           ↓           ↓            ↓              ↓           ↓
  愿景使命    70%原则     OKR对齐       董事会报告      竞品分析    快速响应
```

## 核心决策规则（始终激活）

**3优先级原则**：同时在做的重点不超过3个，超过=没有重点。

**70%信息原则**：有70%信息就要做决定，等100%时机已过。

**优先级排序**：
1. 能影响未来12个月存亡的事
2. 能影响未来3个月目标的事
3. 其他

## 完整工作流

### 阶段一：战略制定

**触发**：年度规划 / 进入新市场 / 业务转型

```
1. 加载战略框架
   skill_view("ceo/strategy")
   → 核心问题：我们在哪里打？凭什么赢？如何持续赢？

2. 战略三问：
   - 我们的不可替代性是什么？
   - 未来3年最大的外部变化是什么？
   - 现在的资源配置是否与战略匹配？

3. 输出：战略一页纸（1页，不能更多）
```

---

### 阶段二：决策执行

**触发**：面临重要选择 / 需要取舍

```
1. 加载决策框架
   skill_view("ceo/decisions")
   → 反向思考：如果这个决定错了，最坏结果是什么？能承受吗？
   → 决策日志：记录每次重要决定的假设和预期

2. 快速决策原则：
   - 可逆决定 → 快速试，成本低
   - 不可逆决定 → 慢思考，寻求第二意见

3. 写入 TASKLOG.md
```

---

### 阶段三：董事会与投资人管理

**触发**：董事会会议前 / 融资 / 投资人沟通

```
1. 加载董事会框架
   skill_view("ceo/board")
   → 董事会汇报结构：核心指标→风险→需要决策的事→下季度计划
   → 黄金原则：不要等董事会才同步坏消息

2. 融资叙事三要素：
   - 为什么现在？（市场时机）
   - 为什么是我们？（差异化）
   - 为什么这个数字？（资金用途和里程碑）
```

---

### 阶段四：竞争情报

**触发**：竞品有新动作 / 进入新市场 / 客户流失

```
1. 加载竞争情报框架
   skill_view("ceo/competitive-intel")
   → 监测维度：产品/定价/营销/人才/融资

2. 竞争应对原则：
   - 不要为竞争对手的决定花超过20%精力
   - 关注客户，而非竞争对手
   - 有差异化才是真正的护城河

3. 定期更新：每季度做一次竞争格局扫描
```

---

### 阶段五：组织与文化

**触发**：管理层问题 / 公司规模跃迁 / 文化危机

```
1. 加载人才框架
   skill_view("ceo/people")
   → 关键高管判断：Is this person still the right person for this role?
   → 文化信号：员工在做什么 vs 说什么

2. CEO时间配置原则：
   - 70% 核心业务（客户/产品/销售）
   - 20% 团队与文化
   - 10% 外部（投资人/合作）
```

---

## 主动提醒规则

- 用户说「要做决定」→ 主动用70%原则+反向思考框架辅助决策
- 用户说「竞争对手」→ 主动问：是要分析竞争格局还是制定应对策略？
- 用户说「董事会」→ 主动问：需要准备董事会材料吗？
- 用户说「方向不对」→ 加载战略框架，重新三问
- 季度末 → 主动提醒：季度复盘 + 下季度OKR对齐时间
