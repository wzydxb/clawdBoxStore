# 被动蒸馏提取器（对话信号）

## 任务

从本次对话中提取用户的**个人信号**，静默追加到分身文件。

**原则：宁缺毋滥。只记录有实质信息量的信号，闲聊和指令类对话不记录。**

---

## 提取标准

### 值得记录的信号

| 信号类型 | 举例 |
|----------|------|
| 价值判断 | "我觉得这种做法没意义"、"效率比形式重要" |
| 习惯偏好 | "我一般睡前看完"、"我不喜欢开大会" |
| 情绪反应 | 对某件事明显烦躁/兴奋/冷淡 |
| 身份信息 | 提到职业、城市、家庭情况、某段经历 |
| 说话风格 | 口头禅、惯用句式、反复出现的表达方式 |
| 决策模式 | 做决定时的典型思路 |
| 关系信息 | 提到某个人时的态度和描述 |
| 痛点/抱怨 | 反复提到的烦恼或不满 |

### 不记录的内容

- 纯粹的任务指令（"帮我查一下..."、"整理这个..."）
- 对话中的客套话
- 已经在分身文件中有记录的信息（除非有新细节）
- 不确定是不是用户本人观点的内容

---

## 提取流程

### Step 1：快速扫描本次对话

回顾本次对话，判断：
- 用户说了多少句"真正属于自己"的话？
- 有没有情绪波动？有没有偏好表达？有没有自我披露？

如果全是任务指令，没有个人信号 → **直接跳过，不写入**

### Step 2：分类提取

提取信号，分类到以下维度：

**self.md 类（事实/记忆/价值观）**：
```
维度：[核心价值观 / 生活习惯 / 重要记忆 / 人际关系 / 成长轨迹]
内容：[具体描述，引用原话]
置信度：[高/中/低]
```

**persona.md 类（性格/口吻/情绪模式）**：
```
维度：[说话风格 / 情感模式 / 决策模式 / 自我对话 / 人际行为]
内容：[具体描述，引用原话]
置信度：[高/中/低]
```

### Step 3：写入判断

只有**置信度为"高"或"中"**的信号才写入文件。

低置信度的信号记录在 `self.md` 末尾的 `## 待确认信号` 区块，等积累 3 次以上一致信号后再升级为正式记录。

---

## 写入格式

追加到 `self.md` 或 `persona.md` 对应章节末尾：

```markdown
<!-- [对话蒸馏·{日期}] -->
- {具体内容}（原话：「{引用}」）
```

如果对应章节不存在，追加到文件末尾并新建章节标题。

---

## 重要约束

- **静默执行**：不向用户报告"我记录了什么"，不打断对话
- **不修改已有内容**：只追加，不覆盖
- **冲突处理**：新信号与已有记录矛盾时，用 `[⚠️ 待确认冲突]` 标注，不擅自修改
- **分身不存在时**：如果用户还没有创建分身，将信号暂存到 `/tmp/yourself_pending.md`，等用户创建分身时一次性导入
