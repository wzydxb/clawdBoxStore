# PPT-Nano 技能 — 多风格 PPT 生成

## 1. 触发与默认模式

用户说「ppt-nano-master」「生成 PPT」「做 PPT」等关键词时，默认进入**草稿直出模式**：
- 最多只提问一次，而且问题要短
- 不等回复也先出一版
- 先做首页主视觉，再做正文页
- 详细流程不在对外回复里展开

只有用户明确要求逐步确认时，才切换到严格确认流程。

### 草稿直出模式下的唯一短问句

可用短问句示例：
> 我先按这个主题出首页主视觉和一版 10 页内的结构稿，可以吗？

如果用户未回复：
- 直接继续，不再追问。

## 3. 页面结构规划层

收到主题或文案后，先给出**结构化页表**，再继续生成。

格式：

> 📋 梳理如下，请确认：
>
> | 页码 | 页面类型 | 参考图 | 内容摘要 |
> |------|---------|--------|---------|
> | 封面 | 首页 | cover.jpg | 主标题 / 副标题 |
> | P1 | 内容页 | content.jpg | 标题 / 要点摘要 |
> | P2 | 图表页 | chart.jpg | 标题 / 数据摘要 |
> | ... | ... | ... | ... |
>
> 共 X 页，默认我先按这个结构做草稿版。

### 页面类型判断规则

- 有时间轴 / 流程 / 组织架构 → **导航页** `navigation.jpg`，优先做成流程图 / 时间轴 / 箭头关系图
- 有数据表格 / 数字对比 / 趋势 → **图表页** `chart.jpg`，优先做成图表或表格
- 有场景故事 / 应用分层 / 场景分类 → 优先做成**卡片式内容页**
- 信息密集但适合分块展示 → 优先拆成**卡片式内容页**
- 纯文字列表 / 要点 → **文字页** `text.jpg`（= `content.jpg`），但仍优先压缩成少量核心点
- 封面 → **首页** `cover.jpg`，且首页背景图必须来自当次主题生成的主视觉图
- 结尾 → **尾页** `closing.jpg`
- 不同题材要重新判断页面结构，不允许直接复用上一套数据和图表模板

### 重要约束

- 数据页只能使用当次任务真实输入或当次检索得到的数据
- 没有真实数据时，必须明确标注“示意数据/待补数据”
- 不允许伪装成真实数据

默认直接进入后续生成；只有用户明确要求“先确认再生成”时，才等待。

## 4. 首页主视觉与整套风格定义层

这一层是整个技能最重要的部分。

### 首页必须包含的固定元素

首页默认必须包含：
1. 与用户主题**严格相关**的背景主视觉图
2. 大标题
3. 与主题气质匹配的标题字体表现
4. 如有必要，再补一行副标题或场景说明

### 核心规则

1. 先根据**用户场景调研后的核心主题集合**，生成首页主视觉的生图提示词
2. 再生成首页主视觉图
3. 将这张图设置为首页背景图
4. 首页大标题与字体样式必须和主题气质一致
5. 由首页背景图和标题风格反推出整套 PPT 的：
   - 色板
   - 卡片样式
   - 图表样式
   - 图标样式
   - 页面气质
   - 标题层级
6. 没有首页主视觉图时，不要进入正文页生成
7. 后续内容页必须跟首页主视觉保持一致

### 视觉原则

- 风格由主题决定，不是固定套一个科技风
- 主题必须和首页图严格相关，不能跑偏
- 医疗就要专业干净，商务就要稳重利落，技术就要更有科技感
- 不同题材不能硬套同一套假数据和图表模板

## 总入口（闭环执行）

当用户已经给出主题、调研结果或大纲时，优先使用总入口脚本跑完整闭环：

```bash
/root/.openclaw/venvs/ppt/bin/python {baseDir}/scripts/run_pipeline.py \
  --theme "{主题}" \
  --research-file "{调研文件或整理后的文本文件}" \
  --slides 8 \
  --style whiteboard \
  --output-dir "{workspace}/ppt_outputs/{任务名}" \
  --output-pptx "{workspace}/ppt_outputs/{任务名}/final.pptx"
```

这个总入口会自动完成：
1. 提炼主题与叙事主线
2. 规划页面结构
3. 选择组件类型
4. 逐页生成图片
5. 合成最终 PPT

如果用户只要单页草稿或你只想单独试某一页，再退回逐页命令模式。

## 5. 组件驱动的页面规划层

正文页不是先定死模板，而是先看内容，再决定组件。

### 组件选择规则

- 数据类内容 → 优先用图表 / 表格 + 简短结论文字
- 趋势变化 → 优先用折线图
- 对比差异 → 优先用柱状图、条形图或对照表
- 占比结构 → 只在合适时用饼图 / 环形图
- 场景分类 / 应用分层 / 能力拆解 → 优先用卡片矩阵
- 流程 / 路径 / 审批 / 执行顺序 → 优先用流程图 / 时间轴 / 箭头关系图
- 核心结论 / 建议 / 行动项 → 优先用结论卡片或重点结论页
- 尾页默认是**致谢页**

### 页面规划硬规则

1. 每页的内容先决定组件，再决定版式
2. 不允许把所有内容都塞成纯文字页
3. 图表、卡片、流程图的视觉语言必须和首页保持一致
4. 图表和表格只承载当次任务的真实数据或明确标注的示意数据

## 6. 页面生成执行层

### 依赖检查（首次使用或不确定时执行）

当前页面生成不再依赖外部 OpenAI 兼容图片 API，而是走系统底层的 `clawdbox-image-gen` 工具。

```bash
/root/.openclaw/venvs/ppt/bin/python -c "import PIL, pptx; print('ok')" && test -f /root/.openclaw/workspace/skills/clawdbox-image-gen/scripts/gen.py
```

如果失败，先安装或检查：
```bash
/root/.openclaw/venvs/ppt/bin/pip install pillow python-pptx
# 并确认 clawdbox-image-gen skill 已安装到 ~/.openclaw/workspace/skills/clawdbox-image-gen
```

### 每页命令模板

```bash
/root/.openclaw/venvs/ppt/bin/python {baseDir}/scripts/generate_image.py \
  --prompt "{对应前缀}{页面专属提示词}" \
  --filename "{输出文件名}.jpg" \
  --resolution 2K \
  --aspect-ratio 16:9 \
  -i "{参考图绝对路径}"
```

### 关键参数说明

- `-i` 传入参考图（会映射到底层本地工具的编辑模式，当前默认使用首张参考图）
- `--resolution` 固定用 `2K`（支持 `2K` / `3K`，内部会映射到底层工具的尺寸）
- `--aspect-ratio` 固定用 `16:9`
- `--filename` 必须以 `.jpg` 结尾（脚本会把底层工具生成的图片统一转成 JPEG）
- 每次调用只生成一页

### 提示词来源

每个风格在 `{baseDir}/styles/styles.json` 中有对应的 `style_prompts` 配置。生成时必须读取对应风格提示词，并与当次页面内容结合，不允许直接机械复用上一轮的提示词。

### 白板板书风格专用前缀（向后兼容）

#### 通用页（内容页 / 图表页 / 导航页 / 尾页）
```
按参考图出图，白板背景和整体版式保留不变，白板上的所有原有文字和图案全部清除，按以下文案重新设计内容。中文必须是马克笔 shufa 笔触风格，配图手绘风格。Chinese text MUST use thick shufa-style ink brush marker calligraphy — bold chunky strokes, NOT printed font. 颜色规则：黑色用于主标题和正文，蓝色用于副标题、数据标注和补充信息，红色用于强调词、结论句和圆圈高亮。配图颜色按内容用红蓝马克笔适配。
```

#### 封面页（cover.jpg 专用，必须使用此版本）
```
按参考图出图，白板背景和整体版式保留不变，白板上的所有原有文字和图案全部清除，不保留任何已有标题或说明文字。按以下文案重新设计封面内容，主标题突出显示，副标题用蓝色马克笔书写。封面版面要充分利用，除主标题和副标题外，可添加与主题相关的手绘装饰图案、关键词标注、或视觉分割线，确保白板面积利用率不低于 60%。中文必须是马克笔 shufa 笔触风格，配图手绘风格。Chinese text MUST use thick shufa-style ink brush marker calligraphy — bold chunky strokes, NOT printed font. DO NOT reproduce any text from the reference image. 颜色规则：黑色用于主标题，蓝色用于副标题和说明文字，红色用于强调词和圆圈高亮。
```

**其他风格：** 读取 `{baseDir}/styles/styles.json` 中对应风格的 `style_prompts` 配置。

## 8. 合成与后处理层

### 合成命令

```bash
/root/.openclaw/venvs/ppt/bin/python {baseDir}/scripts/build_pptx.py \
  -i "slide1.jpg" "slide2.jpg" "slide3.jpg" \
  -o "{workspace}/你的文件名.pptx"
```

### `build_pptx.py` 说明

- 已内置跨平台支持（Windows / macOS / Linux）
- 当前盒子环境统一使用 `/root/.openclaw/venvs/ppt/bin/python` 调用
- 中文路径乱码问题已在脚本内部处理，无需手动设置环境变量

### 参数说明

| 参数 | 说明 |
|------|------|
| `-i img1.jpg img2.jpg ...` | 图片路径列表，按页序排列 |
| `--dir ./某目录/` | 扫描目录下所有 jpg（按文件名字母序） |
| `-o output.pptx` | 输出路径，默认 `output.pptx` |
| `--width` / `--height` | 幻灯片尺寸（英寸），默认 13.33 × 7.5（16:9） |

### 机器可读输出

脚本最后一行输出：
```text
PPTX:/path/to/file.pptx
```

### 对外回复规则

- 合成完成后，对用户的文字回复要极简
- 只需简短说明“已生成/已更新/已合成”
- 不要重复展开每一页的详细内容或分析过程，除非用户明确要求看过程说明

## 10. 常见错误（禁止）

- ❌ 一上来就长篇追问
- ❌ 没有首页主视觉就直接做正文页
- ❌ 首页背景图和主题不严格相关
- ❌ 首页风格和正文页风格割裂
- ❌ 数据页只有大段文字，没有图表/表格
- ❌ 流程页不用流程图，只用段落硬写
- ❌ 不同题材复用同一套组件模板和假数据
- ❌ 不传参考图（`-i`）或混用参考图
- ❌ 封面页使用通用前缀（如该风格有封面专用前缀）
- ❌ 省略提示词前缀
- ❌ 用错参考图（如内容页用了 `chart.jpg`）
- ❌ 忽略 `styles.json` 中的风格配置
- ❌ 不同题材复用同一套假数据和图表模板
- ❌ 把信息密集页直接做成大段文字页
