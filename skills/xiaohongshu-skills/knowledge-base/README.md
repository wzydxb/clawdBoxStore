# 小红书运营知识库

总览入口。Agent 开始任务前先看这里"我们已经知道什么"，结束任务后把新结论沉淀到对应目录。

## 目录结构

```
knowledge-base/
  README.md              # 总览入口
  accounts/              # 账号定位、账号体检、竞品分析
  topics/                # 选题候选、争议点、标题骨架
  patterns/              # 爆款结构、封面模板、互动钩子
  actions/               # 发布、回复、抓取等动作记录
  reviews/               # 复盘、有效/无效原因、调整建议
```

## 使用规则

- **任务前**：先读本文件，再按目录搜索最相关的记录。
- **任务中**：遇到高价值结论，先记临时摘要。
- **任务后**：至少补一条结构化记录到对应目录。

## 文件命名

按"日期 + brief"命名，方便检索：

```
accounts/2026-04-02-drama-watch-positioning.md
topics/2026-04-02-camping-argument-hooks.md
patterns/2026-04-02-confirmation-comment-hook.md
actions/2026-04-02-publish-camping-note.md
reviews/2026-04-02-reply-flow-retrospective.md
```

## 记录最小标准

每条记录至少包含：

```yaml
id: 2026-04-02-brief-description
type: account | topic | pattern | action | review
status: active | deprecated | experimental
created_at: ISO8601
source: 来源（URL、账号名、笔记 ID）
summary: 一句话结论
evidence: 证据
next_action: 下一步怎么用
```
