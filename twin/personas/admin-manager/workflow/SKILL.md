# 行政经理工作流

## 工作流全景

```
采购需求 → 供应商筛选 → 比价谈判 → 合同签署 → 资产入库 → 费用管控 → 制度合规
    ↓           ↓           ↓          ↓          ↓          ↓          ↓
  需求收集    资质审查    三方比价    条款审核    台账登记    预算对比    定期检查
```

---

## 完整工作流

### 阶段一：采购管理

**触发**：部门提交采购需求 / 合同到期续签 / 供应商评估

```
1. 加载采购框架
   skill_view("admin-manager/procurement")
   → 关键问题：
     - 需求是否合理？有无替代方案？
     - 现有供应商能否满足？
     - 预算是否充足？

2. 执行采购流程：
   - 需求确认 → 供应商筛选 → 三方比价 → 谈判 → 合同签署
   - 写入 TASKLOG.md：| 采购-[物品/服务] | 进行中 | [日期] | 预算X万 |
```

---

### 阶段二：资产管理

**触发**：新资产入库 / 年度盘点 / 资产报废申请

```
1. 加载资产管理框架
   skill_view("admin-manager/asset-management")
   → 资产台账更新
   → 折旧计算
   → 盘点差异处理

2. 资产全生命周期：
   - 入库登记 → 领用分配 → 日常维护 → 盘点核查 → 报废处置
```

---

### 阶段三：费用管控

**触发**：月度预算审核 / 费用超支预警 / 年度预算编制

```
1. 加载费用管控框架
   skill_view("admin-manager/expense-control")
   → 预算执行率分析
   → 超支原因定位
   → 优化建议

2. 费控闭环：
   - 预算编制 → 执行跟踪 → 偏差分析 → 调整优化
   - 每月出具行政费用分析报告
```

---

### 阶段四：制度合规

**触发**：新制度发布 / 制度到期修订 / 合规检查

```
1. 加载制度管理框架
   skill_view("admin-manager/policy-compliance")
   → 制度清单盘点
   → 版本管理
   → 合规检查清单

2. 制度管理原则：
   - 每项制度有明确的责任人和修订周期
   - 版本号规范：V1.0（首发）→ V1.1（小修）→ V2.0（大改）
   - 新制度发布后 7 天内完成全员宣贯
```

---

### 阶段五：差旅安排

**触发**：出差申请 / 团队外出 / 客户来访接待

```
1. 加载差旅框架
   skill_view("admin-manager/travel-logistics")
   → 差旅标准对照
   → 行程规划
   → 费用预算

2. 差旅全流程：
   - 出差申请 → 行程规划 → 预订（机票/酒店/车辆）→ 出行 → 报销核对
```

---

### 阶段六：办公运营

**触发**：办公环境问题 / 年度行政规划 / 跨部门协调

```
1. 加载行政顾问框架
   skill_view("admin-manager/admin-advisor")
   → 办公空间规划
   → 年度行政预算
   → 行政数字化

2. 日常运营：
   - 办公环境维护（保洁/绿植/设备）
   - 会议室管理
   - 快递收发
   - 来访接待
```
